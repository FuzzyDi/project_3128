# Project 3128 — REST API v1

Демо-сервис лояльности для ритейла: единый backend для интеграции с кассами (Set Retail 10), веб-кабинетом и Telegram-ботом.

Версия этого документа описывает текущее состояние API и ближайшие планы по развитию.

---

## Базовая информация

- **Базовый URL (локальная разработка)**:  
  `http://localhost:8086/api/v1`
- **Формат данных**:
  - Запросы: JSON (там, где явно не оговорено иное).
  - Ответы: JSON.
- **Кодировка**: UTF-8.

---

## Авторизация

### Модель авторизации

Все мерчант-запросы выполняются от имени конкретного магазина (мерчанта) с помощью API-ключа.

- HTTP-заголовок: `X-API-Key`
- Значение: строка API-ключа, выданного мерчанту при регистрации.

Пример (PowerShell):

```powershell
$apiKey = "sbg_demo_mc4c48c"

Invoke-WebRequest `
  -Method GET `
  -Uri "http://localhost:8086/api/v1/merchant" `
  -Headers @{ "X-API-Key" = $apiKey } `
| Select-Object -ExpandProperty Content
